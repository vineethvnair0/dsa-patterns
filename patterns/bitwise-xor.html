Pattern 22: Bitwise XOR

Bit Manipulation

Pattern Overview

The XOR (exclusive OR) bitwise operation is a powerful tool for solving specific algorithmic problems with exceptional efficiency. This pattern leverages unique mathematical properties of XOR to achieve solutions with O(1) space complexity.

The XOR pattern is particularly effective for problems involving finding unique elements, detecting duplicates, and manipulating binary representations. Understanding these properties is crucial for technical interviews at major tech companies.

Problem: Single Number

Given a non-empty array of integers where every element appears twice except for one element which appears only once, find that single element.

Example:
Input: nums = [2, 2, 1]
Output: 1

How XOR works:

  • Identity Property: a ⊕ a = 0 (any number XOR'd with itself equals 0)
  • Zero Property: a ⊕ 0 = a (any number XOR'd with 0 equals itself)
  • Commutative Property: a ⊕ b = b ⊕ a (order doesn't matter)
  • Associative Property: (a ⊕ b) ⊕ c = a ⊕ (b ⊕ c)
  • When we XOR all numbers together, duplicate pairs cancel out to 0
  • The single number remains: 2 ⊕ 2 ⊕ 1 = 0 ⊕ 1 = 1

Example Problem

Problem: Single Number

Given a non-empty array of integers where every element appears twice except for one element which appears only once, find that single element.

Example 1:
Input: nums = [2, 2, 1]
Output: 1
Example 2:
Input: nums = [4, 1, 2, 1, 2]
Output: 4
Example 3:
Input: nums = [1]
Output: 1

Solution Code

// Single Number using XOR public static int singleNumber(int[] nums) { int result = 0; // XOR all numbers together // Pairs will cancel out (a ⊕ a = 0) // Only the single number remains for (int num : nums) { result ^= num; } return result; }

Solution Explanation

Step-by-Step Breakdown:

The XOR approach solves this problem efficiently using the following properties:

  1. Initialize result to 0: Start with result = 0. This is our identity element for XOR.
  2. XOR all elements: Iterate through the array and XOR each element with the result:
    • XOR has the property: a ⊕ a = 0 (any number XORed with itself equals 0)
    • XOR has the property: a ⊕ 0 = a (any number XORed with 0 equals itself)
    • XOR is commutative and associative (order doesn't matter)
  3. Return result: After XORing all elements, pairs cancel out to 0, leaving only the single number.

Key Insight: For array [2, 2, 1], the calculation is: 0 ⊕ 2 ⊕ 2 ⊕ 1 = 0 ⊕ 0 ⊕ 1 = 1. The two 2's cancel each other out (2 ⊕ 2 = 0), leaving just 1.

Complexity Analysis

Time Complexity: O(n) | Space Complexity: O(1)

Time Complexity: O(n)

  • Single pass through the array
  • Each element is processed exactly once
  • XOR operation takes constant time O(1)
  • Total: O(n) where n is the number of elements

Space Complexity: O(1)

  • Only uses a single variable (result) to store the XOR value
  • No additional data structures needed
  • Space used is constant regardless of input size

Common Use Cases

  • Finding single number in array of duplicates
  • Finding two missing numbers
  • Detecting duplicate in array
  • Finding different character in strings
  • Swapping variables without temporary variable
  • Checking if two integers have opposite signs

Key Concepts

  • Self-Cancellation: XOR'ing a number with itself always results in 0
  • Identity Element: XOR'ing with 0 leaves the number unchanged
  • Order Independence: XOR operations can be performed in any order
  • Space Efficiency: Typically requires O(1) extra space
  • Bit-Level Operations: Works directly with binary representations
  • Applications: Finding unique elements, swapping values, detecting differences

Hard 1707. Maximum XOR With an Element From Array ▼

Description: You are given an array nums consisting of non-negative integers. You are also given a queries array, where queries[i] = [xi, mi]. The answer to the ith query is the maximum bitwise XOR value of xi and any element of nums that does not exceed mi. In other words, the answer is max(nums[j] XOR xi) for all j such that nums[j] <= mi. If all elements in nums are larger than mi, then the answer is -1.

Example 1:

Input: nums = [0,1,2,3,4], queries = [[3,1],[1,3],[5,6]]
Output: [3,3,7]
Explanation:
1) 0 and 1 are the only two integers not greater than 1. 0 XOR 3 = 3 and 1 XOR 3 = 2. The larger is 3.
2) 1 XOR 2 = 3.
3) 5 XOR 2 = 7.

Output: [15,-1,5]

Constraints:

  • 1 <= nums.length, queries.length <= 10^5
  • queries[i].length == 2
  • 0 <= nums[j], xi, mi <= 10^9

Hard 1938. Maximum Genetic Difference Query ▼

Description: There is a rooted tree consisting of n nodes numbered 0 to n - 1. Each node's number denotes its unique genetic value (i.e., the genetic value of node x is x). The genetic difference between two genetic values is defined as the bitwise-XOR of their values. You are given the integer array parents, where parents[i] is the parent for node i. If node x is the root of the tree, then parents[x] == -1. You are also given the array queries where queries[i] = [nodei, vali]. For each query i, find the maximum genetic difference between vali and pi, where pi is the genetic value of any node that is on the path between nodei and the root (including nodei and the root). More formally, you want to maximize vali XOR pi.

Example 1:

Input: parents = [-1,0,1,1], queries = [[0,2],[3,2],[2,5]]
Output: [2,3,7]
Explanation: The queries are processed as follows:
- [0,2]: The node with the maximum genetic difference is 0, with a difference of 2 XOR 0 = 2.
- [3,2]: The node with the maximum genetic difference is 1, with a difference of 2 XOR 1 = 3.
- [2,5]: The node with the maximum genetic difference is 2, with a difference of 5 XOR 2 = 7.

Example 2:

Input: parents = [3,7,-1,2,0,7,0,2], queries = [[4,6],[1,15],[0,5]]
Output: [6,14,7]

Constraints:

  • 2 <= parents.length <= 10^5
  • 0 <= parents[i] <= parents.length - 1 for every node i that is not the root
  • parents[root] == -1
  • 1 <= queries.length <= 3 * 10^4
  • 0 <= nodei <= parents.length - 1
  • 0 <= vali <= 2 * 10^5

    - (1, 3): nums[1] XOR nums[3] = 3

Example 2:

Input: nums = [9,8,4,2,1], low = 5, high = 14
Output: 8
Explanation: All nice pairs (i, j) are as follows:
    - (0, 2): nums[0] XOR nums[2] = 13

    - (1, 3): nums[1] XOR nums[3] = 10
    - (1, 4): nums[1] XOR nums[4] = 9
    - (2, 3): nums[2] XOR nums[3] = 6
    - (2, 4): nums[2] XOR nums[4] = 5
