Pattern 1: Sliding Window

Arrays & Strings

Pattern Overview

A technique used to perform operations on a specific window size of an array or string. The window "slides" through the data structure to find optimal solutions. This pattern is extremely efficient for problems involving contiguous subarrays or substrings.

The sliding window pattern helps avoid nested loops and reduces time complexity from O(n²) or O(n*k) to O(n) in many cases.

Problem: Maximum Sum Subarray of Size K

Given an array of positive numbers and a positive number 'k', find the maximum sum of any contiguous subarray of size k.

Example:
Input: array = [2, 1, 5, 1, 3, 2], k = 3
Output: 9 (subarray [5, 1, 3])

How it works:

  • Calculate the sum of the first k elements to form the initial window
  • Slide the window one position to the right
  • Instead of recalculating the entire sum, subtract the element leaving the window and add the new element entering
  • Track the maximum sum seen across all window positions
  • This optimization reduces time complexity from O(n*k) to O(n)
  • Example: Window moves from [2,1,5] → [1,5,1] → [5,1,3] → [1,3,2]

Example Problem

Problem: Maximum Sum Subarray of Size K

Given an array of positive numbers and a positive number 'k', find the maximum sum of any contiguous subarray of size k.

Example 1:
Input: arr = [2, 1, 5, 1, 3, 2], k = 3
Output: 9
Explanation: Subarray with maximum sum is [5, 1, 3]
Example 2:
Input: arr = [2, 3, 4, 1, 5], k = 2
Output: 7
Explanation: Subarray with maximum sum is [3, 4]

Solution Code

// Maximum sum subarray of size K public static int maxSubarraySum(int[] arr, int k) { int maxSum = 0, windowSum = 0; // Calculate sum of first window for (int i = 0; i < k; i++) { windowSum += arr[i]; } maxSum = windowSum; // Slide the window for (int i = k; i < arr.length; i++) { windowSum = windowSum - arr[i - k] + arr[i]; maxSum = Math.max(maxSum, windowSum); } return maxSum; }

Solution Explanation

Step-by-Step Breakdown:

The sliding window approach solves this problem efficiently by avoiding recalculation of sums:

  1. Initialize the first window: Calculate the sum of the first 'k' elements. This becomes our initial window sum and maximum sum.
  2. Slide the window: For each subsequent position:
    • Remove the leftmost element from the previous window
    • Add the new rightmost element to the current window
    • Update the maximum sum if current window sum is greater
  3. Return result: After processing all windows, return the maximum sum found.

Key Insight: Instead of recalculating the sum of all k elements for each window (which would be O(n*k)), we reuse the previous sum by subtracting the element that's leaving and adding the element that's entering the window. This reduces computation to O(n).

Complexity Analysis

Time Complexity: O(n) | Space Complexity: O(1)

Time Complexity: O(n)

  • First loop runs k times to calculate initial window sum: O(k)
  • Second loop runs (n-k) times to slide the window: O(n-k)
  • Total: O(k) + O(n-k) = O(n)
  • Each iteration does constant work (one addition, one subtraction, one comparison)

Space Complexity: O(1)

  • Only uses a constant amount of extra space
  • Variables used: maxSum, windowSum (fixed memory regardless of input size)
  • No additional data structures required

Common Use Cases

  • Maximum sum subarray of size K
  • Longest substring with K distinct characters
  • Fruits into Baskets
  • String anagrams
  • Smallest subarray with given sum
  • Longest substring with same letters after replacement

Hard 1438. Longest Continuous Subarray With Absolute Diff Less Than or Equal to Limit ▼

Description: Given an array of integers nums and an integer limit, return the size of the longest non-empty subarray such that the absolute difference between any two elements of this subarray is less than or equal to limit.

Example 1:

Input: nums = [8,2,4,7], limit = 4
Output: 2
Explanation: All subarrays are:
[8] with maximum absolute diff |8-8| = 0 <= 4.
[8,2] with maximum absolute diff |8-2| = 6 > 4.
[8,2,4] with maximum absolute diff |8-2| = 6 > 4.
[8,2,4,7] with maximum absolute diff |8-2| = 6 > 4.
[2] with maximum absolute diff |2-2| = 0 <= 4.
[2,4] with maximum absolute diff |2-4| = 2 <= 4.
[2,4,7] with maximum absolute diff |2-7| = 5 > 4.
[4] with maximum absolute diff |4-4| = 0 <= 4.
[4,7] with maximum absolute diff |4-7| = 3 <= 4.
[7] with maximum absolute diff |7-7| = 0 <= 4.
Therefore, the size of the longest subarray is 2.

Example 2:

Input: nums = [10,1,2,4,7,2], limit = 5
Output: 4
Explanation: The subarray [2,4,7,2] is the longest since the maximum absolute diff is |2-7| = 5 <= 5.

Constraints:

  • 1 <= nums.length <= 10^5
  • 1 <= nums[i] <= 10^9
  • 0 <= limit <= 10^9
