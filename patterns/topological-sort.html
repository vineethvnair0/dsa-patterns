Pattern 19: Topological Sort

Graphs

Pattern Overview

This pattern is essential for solving specific types of algorithmic problems efficiently. It provides a systematic approach to tackle common challenges in coding interviews.

Understanding when and how to apply this pattern will significantly improve your problem-solving skills and interview performance.

Problem: Course Schedule

There are n courses to take (0 to n-1). Some courses have prerequisites. Given the total number of courses and prerequisite pairs, return the ordering of courses you should take to finish all courses.

Example:
Input: numCourses = 4, prerequisites = [[1,0],[2,0],[3,1],[3,2]]
Output: [0,1,2,3] or [0,2,1,3]

How Topological Sort works:

  • Topological Sort orders nodes in a DAG (Directed Acyclic Graph) so all edges go from left to right
  • Calculate in-degree (number of incoming edges) for each node
  • Add all nodes with in-degree 0 to a queue
  • While queue is not empty:
  • 1. Remove a node from queue and add to result
  • 2. For each neighbor, decrement its in-degree
  • 3. If neighbor's in-degree becomes 0, add it to queue
  • Time complexity: O(V + E), Space complexity: O(V)
