Master These 23 Patterns for FAANG/MAANG Interviews

Why Learn These Patterns?

Instead of grinding 1000+ random LeetCode problems, focus on these 23 core patterns that cover 80% of coding interview questions. Each pattern includes detailed explanations, visual diagrams, Java implementations, and curated practice problems organized by difficulty.

1

Sliding Window

Arrays & Strings

Efficiently processes contiguous subarrays/substrings by maintaining a window that slides through the data.

Use Cases:
  • Maximum sum subarray of size K
  • Longest substring with K distinct chars
  • String anagrams

Islands (Matrix Traversal)

Matrices

Traverses 2D matrices to find connected components using DFS/BFS, commonly used for island counting problems.

Use Cases:
  • Number of islands
  • Flood fill
  • Surrounded regions
View Pattern Details
3

Two Pointers

Arrays & Strings

Uses two pointers to iterate through data structure from different ends or at different speeds.

Use Cases:
  • Pair with target sum
  • Remove duplicates
  • Palindrome check
View Pattern Details
4

Binary Search

Arrays

Efficiently searches sorted arrays by repeatedly dividing the search space in half.

Use Cases:
  • Search in rotated array
  • Find peak element
  • Search in 2D matrix
View Pattern Details

Fast & Slow Pointers

Linked Lists

Uses two pointers moving at different speeds to detect cycles and find middle elements.

Use Cases:
  • Detect cycle in linked list
  • Find middle of linked list
  • Happy number problem

Merge Intervals

  • Merge overlapping intervals
  • Insert interval
  • Meeting rooms

7

Cyclic Sort

Arrays

Sorts arrays containing numbers in a given range in O(n) time by placing each number at its correct index.

  • Find missing number
  • Find duplicate number
  • Find all duplicates

9

Top K Elements

Heaps

Finds K largest/smallest elements using heaps for efficient partial sorting.

Use Cases:
  • Kth largest element
  • Top K frequent elements
  • K closest points

10

K-way Merge

Heaps

Merges K sorted arrays/lists using a min-heap to efficiently find the smallest elements.

Use Cases:
  • Merge K sorted lists
  • Kth smallest in M sorted arrays
  • Smallest range in K lists

11

Breadth-First Search (BFS)

Graphs

Explores graphs/trees level by level using a queue, finding shortest paths in unweighted graphs.

Use Cases:
  • Shortest path in grid
  • Word ladder
  • Minimum jumps
View Pattern Details
12

Depth-First Search (DFS)

Graphs

Explores graphs/trees by going deep into branches before backtracking.

Use Cases:
  • Path exists in graph
  • Clone graph
  • Course schedule
View Pattern Details
13

Backtracking

Recursion

Builds solutions incrementally and backtracks when a solution path fails.

Use Cases:
  • Generate permutations
  • N-Queens problem
  • Sudoku solver

Dynamic Programming (DP)

Optimization

  • Fibonacci sequence
  • Longest common subsequence
  • Coin change problem
